# 出貨幫手 V5

LINE 禮物訂單分類、廠商試算表自動上傳工具。  
**所有試算表 ID 都不寫進程式碼**,可以放心放公開 repo。

---

## 架構

- `出貨幫手V5.html`:純前端網頁,可直接打開,或丟 GitHub Pages
- `Code.gs`:Google Apps Script 後端,負責把訂單寫進 Google 試算表

設定資料的存放:

| 設定               | 存在哪                                  | 會跟著程式碼嗎 |
|------------------|--------------------------------------|---------|
| Apps Script 網址   | 瀏覽器 `localStorage`                   | ❌       |
| 6 個試算表網址        | 瀏覽器 `localStorage`                   | ❌       |
| 6 個試算表 ID        | Apps Script 的「指令碼屬性」(Script Properties) | ❌       |

換言之,程式碼裡找不到任何試算表 ID 或網址,可以安心 push 到公開 repo。

---

## 部署步驟

### A. 後端(Google Apps Script)

1. 自己準備 6 個 Google 試算表(可以全空白,新建即可):
   - 雷雕
   - 黑熊
   - 永生花
   - 注意品項
   - 盆景公仔組
   - 問題訂單

   每個試算表的網址都長這樣:
   ```
   https://docs.google.com/spreadsheets/d/<這一段是 ID>/edit
   ```
   把中間那一段 ID 複製下來備用。

2. 開一個新的 Apps Script 專案 → 把 `Code.gs` 全部內容貼進去。

3. 左側齒輪「專案設定」 → 拉到最下方「指令碼屬性」 → 新增 6 個屬性:

   | Key                  | Value(填試算表 ID)        |
   |----------------------|----------------------|
   | `SHEET_ID_LASER`     | 雷雕試算表 ID           |
   | `SHEET_ID_BEAR`      | 黑熊試算表 ID           |
   | `SHEET_ID_FLOWER`    | 永生花試算表 ID         |
   | `SHEET_ID_NOTICE`    | 注意品項試算表 ID       |
   | `SHEET_ID_BONSAI`    | 盆景公仔組試算表 ID     |
   | `SHEET_ID_PROBLEM`   | 問題訂單試算表 ID       |

4. 點右上角「部署」 → 「新增部署」 → 類型選「網路應用程式」:
   - 執行身分:**我**
   - 誰可存取:**任何人**
   - 部署 → 授權 → 取得網址(`https://script.google.com/macros/s/.../exec`)

5. 把這個網址複製下來。

### B. 永生花試算表的欄位

永生花上傳會分成「黃金運」和「粉招福」兩欄,工作表第一列建議填:

```
日期  姓名  地址  電話  備注  訂單編號  黃金數量  粉福數量
```

(沒手動填也沒關係,後端第一次寫入時會自動補表頭。)

其他四個分類試算表的欄位是:

```
日期  姓名  地址  電話  備注  訂單編號  數量
```

問題訂單試算表的欄位是:

```
加入時間  訂單編號  問題類別  備註
```

### C. 前端(網頁)

把 `出貨幫手V5.html` 開起來(可以直接開檔案,也可以丟 GitHub Pages)。

第一次使用:

1. 點右上角「⚙ 設定」
2. 把步驟 A 拿到的 Apps Script 網址貼到第一格
3. 把 6 個試算表網址(完整網址,不是 ID)貼到下面 6 格
4. 按「儲存」

之後就用「↗ 上傳」一鍵把訂單推到對應試算表。

## V5 變更摘要

- ✅ **試算表 ID 全部不寫死**:前端用 `localStorage`、後端用 Script Properties
- ✅ **永生花新增黃金運/粉招福分欄統計**:商品編號相同,用「規格設定」字串判斷顏色
- ✅ **舊資料容錯**:舊版的「粉色/藍色」訂單仍可正確分類(粉色 → 粉招福、藍色 → 黃金運)
- ✅ 永生花試算表自動從 7 欄升級為 8 欄(舊資料保留)

## 永生花顏色判斷規則

訂單的「規格設定」欄位文字 → 統計欄位:

| 規格設定包含關鍵字           | 計入欄位     |
|---------------------|----------|
| 黃金運、黃金              | 黃金數量     |
| 粉招福、粉招、粉福           | 粉福數量     |
| 粉色(舊資料)            | 粉福數量     |
| 藍色(舊資料)            | 黃金數量     |
| 都不符合                | 黃金數量(預設) |
